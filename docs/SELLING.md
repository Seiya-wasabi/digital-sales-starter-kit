## BOOTH

### アカウント準備

- [ ] BOOTHアカウント作成（pixivアカウントが必要）
- [ ] 本人確認（振込のために必要）
- [ ] ショップ設定（ショップ名・説明・アイコン）
- [ ] 振込先銀行口座の登録

### 商品出品

1. [ ] BOOTH管理画面 → 商品 → 新規作成
2. [ ] 商品種別: 「ダウンロード商品」を選択
3. [ ] タイトル: `out/marketplace/booth/{slug}.md` の内容を参照
4. [ ] 説明文: 同ファイルの説明文をコピー
5. [ ] 価格設定（税込価格を入力）
6. [ ] ダウンロードファイルをアップロード
7. [ ] サムネイル画像をアップロード（推奨: 600×600px）
8. [ ] カテゴリ・タグの設定（`config/products.yaml` の `tags_ja` を参照）
9. [ ] 公開設定を「公開」に変更
10. [ ] 商品URLをコピーして `config/products.yaml` の `booth_url` に記入
11. [ ] `pnpm gen` を実行してJSONを更新
12. [ ] GitHubにプッシュ → Cloudflare Pagesが自動デプロイ

### 注意事項

- 手数料は販売金額の22%程度（変動あり）。詳細は公式を確認。
- 振込は月次。最低振込額あり。
- pixivフォロワーへの告知機能を活用する。

3. [ ] 本文: 同ファイルをベースに加筆
4. [ ] 有料エリアの設定: 「有料エリアを設定する」をON
5. [ ] 価格設定（100円〜）
6. [ ] ヘッダー画像の設定
7. [ ] 公開する
8. [ ] 商品URLをコピーして `config/products.yaml` の `note_url` に記入

### デジタルコンテンツファイルの配布方法

noteはファイル添付に制限がある場合があります。以下の方法を検討：

- 有料エリアにファイルのDLリンクを記載（Googleドライブ等）
- または「コンテンツ設定」からファイルをアップロード（対応形式を確認）

### 注意事項

- SEOに強い。記事本文が検索に引っかかりやすい。
- 記事無料部分を充実させることでフォロワーを増やしやすい。
- 手数料は記事価格の10%程度（決済手数料別途）。詳細は公式を確認。

## Gumroad

### アカウント準備

- [ ] Gumroadアカウント作成（メール認証）
- [ ] 支払い先の設定（PayPal or 銀行振込）
- [ ] プロフィール設定

### 商品出品

1. [ ] Gumroad管理画面 → Products → New product
2. [ ] Product type: 「Digital product」を選択
3. [ ] Name: `out/marketplace/gumroad/{slug}.md` の内容を参照
4. [ ] Description: 同ファイルの内容を使用（Markdown対応）
5. [ ] Price: ドル建てで設定（円建ては非推奨）
6. [ ] Cover image をアップロード
7. [ ] コンテンツファイルをアップロード
8. [ ] Tags を設定（`tags_en` を参照）
9. [ ] Publish
10. [ ] 商品URLをコピーして `config/products.yaml` の `gumroad_url` に記入
11. [ ] `pnpm gen` 実行、プッシュ

### 重要: Gumroad Discoverについて

Gumroad Discoverに登録すると、Discover経由の購入は**手数料30%**になる。
直リンク（あなたのLPからの誘導）では**10%**。

- Discoverに依存すると粗利が大幅に下がる
- 自分のSNS・LPから直接誘導することで10%手数料を維持できる
- Discoverへの登録は任意

## Stripe

Stripeは自前LPと組み合わせて使う最も高粗利な構成。

### 前提条件

- [ ] Stripeアカウント作成（本人確認が必要）
- [ ] Stripe本番アカウントの有効化（審査あり）
- [ ] 商品・価格の登録

### 商品・Payment Linkの作成

1. [ ] Stripe Dashboard → 商品 → 商品を作成
2. [ ] 商品名: タイトルを入力
3. [ ] 説明: `out/marketplace/stripe/{slug}.md` を参考に
4. [ ] 価格: 一回限りの支払い、金額を設定
5. [ ] 商品画像をアップロード
6. [ ] Payment Linkを作成（Dashboard → Payment links）
7. [ ] 「購入後にファイルをダウンロード」の設定
   - Stripeでは直接ファイルを配布できないため、購入後URLをGoogle Drive等に設定
   - または Stripe + Zapier/Make でダウンロードURLをメール送信
8. [ ] Payment LinkのURLをコピー → `config/products.yaml` の `stripe_url` に記入
9. [ ] `pnpm gen` 実行、プッシュ

### ファイル配布の方法（Stripe）

Stripeは決済後のファイル配布機能が限定的なため、以下のいずれかを選択：

**方法A（シンプル）**: 購入後のサンキューページにDLリンクを表示
- Payment Link の「確認ページ」にURLを記入
- Google Drive / Dropboxの共有リンクを利用

**方法B（自動化）**: Webhookでメール配信
- Stripe Webhook + Resend/SendGrid でDLリンクをメール送信
- ある程度の技術知識が必要

**方法C（ツール連携）**: Zapier / Make
- Stripe決済完了 → Zapier → メール送信
- ノーコードで実装可能

### Stripe Tax（消費税）対応

- 日本への販売では消費税の対応が必要
- Stripe Taxを有効にすると自動で消費税を計算・収集できる
- 税務処理は会計士に相談を推奨

## 全販路共通 出品後チェックリスト

出品完了後、以下を確認する。

- [ ] `config/products.yaml` の各 `_url` フィールドに正しいURLが入っているか
- [ ] `pnpm gen` を実行したか
- [ ] GitHubにプッシュし、Cloudflare Pagesがビルド・デプロイされたか
- [ ] `/go?product={slug}` にアクセスして正しく販路にリダイレクトされるか
- [ ] `/p/{slug}` の商品詳細ページで「購入する」ボタンが表示されているか
- [ ] `/stats` にアクセスしてクリック計測が動作しているか（要ADMIN_TOKEN）
- [ ] 各販路の商品ページでテスト購入（または友人に確認依頼）

## 価格設定の参考

| 商品種別 | 目安価格帯（円） |
|----------|-----------------|
| チェックリスト・テンプレート（軽量） | 500〜1,980 |

# メトリクスとKPIツリー

最終更新: 2026-03-13

---

## 1. KPIツリー

```
売上（月次）
├── 訪問者数（月間UU）
│   ├── オーガニック検索流入
│   │   └── 対象キーワードランキング
│   ├── SNS流入（X / note）
│   │   └── 投稿インプレッション・エンゲージメント率
│   ├── 参照流入（外部リンク）
│   └── 直接流入
│
├── LP → 購入ページCTR（クリック率）
│   ├── /go クリック数（KV計測）
│   ├── チャネル別クリック内訳
│   └── ref別クリック内訳（lp / detail / bottom / sns等）
│
└── 購入転換率（各プラットフォームで確認）
    ├── チャネル別転換率
    └── 価格別転換率
```

## 2. 計測できるメトリクス（本キット）

### クリック計測（/go エンドポイント）

`/stats` で確認できる項目:

| メトリクス | 説明 |
|-----------|------|
| 総クリック数（N日間） | 購入ボタンの総クリック数 |
| 商品別クリック数 | 商品ごとのクリック数 |
| チャネル別クリック数 | booth / note / gumroad / base / stripe 別 |
| ref別クリック数 | lp / detail / bottom / sns / email 等 |
| 日別クリック数 | 日付ごとの推移（最大90日） |

### 計測できないもの（PIIなし設計のため）

- ユーザーID・セッションID
- IPアドレス
- ユーザーエージェント（デバイス種別）
- 購入転換率（プラットフォームで別途確認）

## 3. 推奨KPIと目標値（参考）

| KPI | 初月目標 | 3ヶ月目標 | 6ヶ月目標 |
|-----|---------|---------|---------|
| 月間LPクリック | 50 | 200 | 500 |
| 月間/go クリック（購入ボタン） | 10 | 50 | 150 |
| 購入転換率（各プラットフォーム） | 3% | 5% | 8% |
| 月間売上 | 1万円 | 5万円 | 15万円 |

※上記は参考値。商品・市場・プロモーション次第で大きく変わります。

## 4. 計測の読み方

### クリック数が少ない → 流入問題

対策:
- SNS投稿頻度を上げる
- SEO記事を書く
- note / BOOTHでの告知を増やす

### クリック数は多いが売れない → 転換率問題

対策:
- 価格設定の見直し
- 商品説明の改善（ベネフィットを明確に）
- FAQの充実（不安解消）
- 社会的証明の追加（レビュー等）

### 特定のチャネルだけクリックが少ない → チャネル設定問題

対策:
- そのチャネルのURLが正しく設定されているか確認
- 販路ページの商品説明が弱い可能性

### refが「lp」に偏っている → ファネルの出口が偏っている

対策:
- 商品詳細ページ（ref=detail）へのトラフィックを増やす
- FAQページからの誘導を強化

## 5. 実験テンプレート

### 実験1: 価格変更実験

**目的**: 価格が転換率に与える影響を測る

```
実験期間: 2週間
仮説: 9,800円→7,800円に下げると転換率が2倍になる
計測: 各プラットフォームの売上レポート + /stats のクリック数
成功基準: 売上合計が変更前を上回る（粗利ベース）
```

### 実験2: CTA文言変更実験

**目的**: ボタン文言が/goクリック率に与える影響を測る

```
実験期間: 2週間
仮説: 「今すぐ購入」より「無料サンプルを見る」の方がCTRが高い
計測: /stats の ref別クリック数
成功基準: CTRが10%以上改善する
実装方法: Astroコンポーネントの文言変更 → git push → 自動デプロイ
```

### 実験3: チャネル告知比較

**目的**: どのSNSからの流入が最も転換するか

```
実験期間: 1ヶ月
仮説: note記事 > X(Twitter) > 直接流入 の順で転換率が高い
計測: /go?ref=note / /go?ref=twitter / /go?ref=direct のクリック数比較
実装方法: SNS投稿時にrefパラメータを付与
```

## 7. Cloudflare Web Analytics との組み合わせ

本キットのクリック計測に加えて、Cloudflare Web Analytics（無料）を使うと以下が分かる:

- ページ別PV数
- 訪問者の国別割合
- デバイス種別（デスクトップ/モバイル）
- リファラー（どこから来たか）

**設定方法**:
1. Cloudflare Dashboard → Web Analytics → サイトを追加
2. 発行されたスクリプトタグをAstroの `<head>` に追加
3. プライバシーファーストでCookieなし・IPなし

Cloudflare Web AnalyticsはCookieを使わないため、Cookieバナーが不要。

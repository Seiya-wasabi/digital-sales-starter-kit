## 1. 各チャネルの概要と向き不向き

### BOOTH

**概要**: pixiv運営のクリエイター向けマーケットプレイス。同人・デジタルコンテンツに強い。

| 項目 | 詳細 |
|------|------|
| 手数料（目安） | 22%前後（変動あり） |
| 対象通貨 | 日本円 |
| 支払サイクル | 月次 |
| 強み | pixivフォロワーへの告知、日本語圏、クリエイター文化 |
| 弱み | 手数料が高め、海外向けには弱い |
| 向いているコンテンツ | 同人系、クリエイティブ、デジタルツール |

**攻略ポイント**:
- pixivで定期的に作品を投稿し、フォロワーを増やしてからBOOTHに誘導するのが効果的
- ショップの「お知らせ」機能を活用してフォロワーに新商品を告知
- タグは日本語で詳細に設定する（検索流入に影響）
- 商品ページのサムネイルは印象が強いものを使う（競合が多い）

---

### note

**概要**: テキストコンテンツと販売が一体化したプラットフォーム。

| 項目 | 詳細 |
|------|------|
| 手数料（目安） | 10%前後（決済手数料別） |
| 対象通貨 | 日本円 |
| 支払サイクル | 月次 |
| 強み | SEO、長文コンテンツ、フォロワーへの告知、無料〜有料のグラデーション |
| 弱み | ファイル配布機能に制限がある場合がある |
| 向いているコンテンツ | ハウツー記事、ノウハウ集、テンプレート（記事形式） |

**攻略ポイント**:
- 無料記事を多数投稿してフォロワーを獲得し、有料コンテンツへ誘導する「バーター戦略」が有効
- 記事の無料部分でベネフィットを明確に伝える
- サムネイルはCanva等でデザインし、目を引くものにする
- noteマガジンでコンテンツをまとめると単価を上げやすい

---

### Gumroad

**概要**: 英語圏を中心としたデジタルコンテンツ販売プラットフォーム。

| 項目 | 詳細 |
|------|------|
| 手数料（直リンク） | 10% |
| 手数料（Discover経由） | 30% |
| 対象通貨 | USD（推奨） |
| 支払サイクル | 週次 |
| 強み | 英語圏向け、シンプルな設定、Discover機能 |
| 弱み | 日本語圏での認知が低い |
| 向いているコンテンツ | テンプレート、デザイン素材、英語コンテンツ |

**攻略ポイント**:
- **Gumroad Discoverの手数料に注意**: Discoverタブに掲載されると認知が広がるが、経由購入は30%の手数料がかかる。自前の告知（SNS・LP）から直接購入してもらうことで10%の手数料を維持できる。
- USD価格設定にする（Gumroadは海外向けが強い）
- 英語の商品説明を用意する
- Product Updatesを活用して既存購入者にアップデートを通知できる

---

### BASE

**概要**: 国内EC向けプラットフォーム。実物・デジタル両方に対応。

| 項目 | 詳細 |
|------|------|
| 手数料（目安） | サービス利用料3% + 決済手数料3.6%+40円 |
| 対象通貨 | 日本円 |
| 支払サイクル | 月次 |
| 強み | 国内EC向け、デジタルコンテンツ配布App、カスタマイズ性 |
| 弱み | 手数料が複雑、流入は自分で作る必要あり |
| 向いているコンテンツ | 実物+デジタルのセット、国内EC向け |

**攻略ポイント**:
- BASEの「デジタルコンテンツ」Appを活用すると自動配布ができる
- BASEのデザインカスタマイズ機能でショップを整える
- クーポン機能を活用してSNS告知と組み合わせる

---

### Stripe（自前LP連携）

**概要**: 決済インフラ。自前LPと組み合わせると最高の粗利を実現できる。

| 項目 | 詳細 |
|------|------|
| 手数料（国内カード） | 3.6% |
| 手数料（海外カード） | 4.4% |
| 対象通貨 | 多通貨対応 |
| 支払サイクル | 週次（申請で変更可） |
| 強み | 最低手数料、高い信頼性、Stripe Tax対応、柔軟なカスタマイズ |
| 弱み | 設定がやや複雑、ファイル自動配布には別途仕組みが必要 |
| 向いているコンテンツ | すべて（特に高単価商品） |

**攻略ポイント**:
- Payment Linkを使えば技術知識不要でStripe決済が実装できる
- 購入後のサンキューページにダウンロードリンクを設置するのが最もシンプル
- 高単価商品（1万円〜）ほどStripeの手数料メリットが大きくなる
- Stripe Taxを有効にすると消費税の処理が自動化される

## 2. チャネル選択マトリクス

| 状況 | 推奨チャネル |
|------|-------------|
| 日本語圏向け、クリエイターファン | BOOTH |
| ブログ・記事型コンテンツ | note |
| 英語圏向け | Gumroad |
| 実物+デジタルセット | BASE |
| 高粗利を最優先 | Stripe |
| 販売開始直後（全部試す） | 全チャネル併用 |
| 運用を最小化したい | Stripe + 1つの日本語プラットフォーム |

## 3. 手数料比較表（参考値）

10,000円の商品を販売した場合の手取り（目安）:

| チャネル | 手数料 | 手取り（目安） | 備考 |
|----------|--------|---------------|------|
| Stripe | 3.6% | 9,640円 | 最高粗利 |
| note | 10% | 9,000円 | 決済手数料別途 |
| BASE | 6.6%+40円 | 9,300円 | 複合手数料 |
| Gumroad（直リンク） | 10% | 9,000円 | USD換算あり |
| BOOTH | 22% | 7,800円 | 最も手数料高め |
| Gumroad（Discover） | 30% | 7,000円 | Discover経由時 |

※上記は参考値。実際の手数料は公式ページを確認してください。

## 4. スマートルーティングの仕組み

`/go?product=X` にアクセスすると、以下の優先順位でリダイレクト先を決定する。

**日本からのアクセス（デフォルト）**:
1. BOOTH（日本語圏向け最優先）
2. note
3. BASE
4. Gumroad
5. Stripe

**海外からのアクセス**:
1. Gumroad（英語圏向け）
2. Stripe
3. BOOTH
4. note
5. BASE

`?channel=stripe` のように明示的にチャネルを指定することで任意の販路に誘導できる。

## 5. 月1時間運用の最小実行活動

月1回、以下の作業を実施するだけで販売活動を継続できる。

### Week 1（月初）: 数字確認（15分）

- [ ] `/stats?days=30` でクリック数・チャネル別傾向を確認
- [ ] 各販路の売上レポートを確認
- [ ] 顧客からの質問・レビューを確認

### Week 2: コンテンツ更新（15分）

- [ ] SNS投稿テンプレ（`out/social/{slug}/x.md`）から1投稿を実施
- [ ] 商品説明に追記があれば `config/products.yaml` を更新
- [ ] `pnpm gen && pnpm build` で再ビルド・デプロイ

### Week 3: 価格・チャネル見直し（15分）

- [ ] 手数料変更がないか各プラットフォームのお知らせを確認
- [ ] 競合商品の価格調査
- [ ] 必要に応じて価格変更

### Week 4: 振り返りと計画（15分）

- [ ] 今月のKPIを記録（クリック数・売上・転換率）
- [ ] 翌月のSNS投稿計画を立てる
- [ ] 改善点を1つ実施（商品説明の改善、タグ追加など）

**合計: 1時間/月**

## 6. 段階的拡大戦略

### Step 1（0〜1ヶ月目）: 1販路で検証

- まずStripeまたはBOOTHの1つで販売開始
- 最初の10件の反応を分析
- FAQを充実させる

### Step 2（2〜3ヶ月目）: 2〜3販路に拡大

- 反応が良ければ他の販路に出品
- `config/products.yaml` にURLを追加するだけで導線が自動更新される

### Step 3（4ヶ月目以降）: 全販路展開

- 全5販路に展開
- クリック計測のデータから最も転換率の高いチャネルを特定
- 高転換チャネルへのトラフィックを集中させる

## 7. チャネル別失敗パターンと対策

| 失敗パターン | チャネル | 対策 |
|-------------|----------|------|
| Discoverに頼りすぎ | Gumroad | 直リンクからの誘導を強化 |
| フォロワーゼロで出品 | BOOTH/note | まず無料コンテンツでフォロワーを増やす |
| 価格を下げすぎ | 全般 | 競合調査して適切な価格帯に設定 |
| 商品説明が薄い | 全般 | `gen:marketplace` のテンプレをフル活用 |
| 更新を放置 | 全般 | 月1時間ルーチンを確実に実施 |

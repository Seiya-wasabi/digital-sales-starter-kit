# プロジェクトブリーフ — デジタル販売スターターキット

作成日: 2026-03-13
バージョン: 0.1.0

---

## 1. 背景と課題

デジタルコンテンツ（テンプレート・マニュアル・ツール・教材等）を個人が販売しようとすると、以下の課題が発生する。

| 課題 | 詳細 |
|------|------|
| 販路の分散 | BOOTH・note・Gumroad・BASE・Stripeで別々に設定が必要 |
| LP作成コスト | ゼロからHTMLを書くか、有料サービスを使う必要がある |
| 計測の複雑さ | Google Analyticsはプライバシー懸念、Cookieバナー必要 |
| 出品原稿の再作成 | 販路ごとに文章を書き直す手間 |
| 運用の複雑化 | チャネルが増えるほど管理コストが上がる |
| 法的ページの不備 | 特商法・利用規約・プライバシーポリシーの準備が面倒 |

本キットはこれらの課題をワンセットで解決する実行テンプレートである。

---

## 2. プロジェクトの目標

### 主目標

「設定ファイル（YAML）を編集するだけで、デジタルコンテンツ販売の主要インフラが揃う」状態を実現する。

### 副目標

- 月1時間以内の運用工数で維持できること
- 月額コスト$0から開始できること（Cloudflare無料枠内）
- プログラミング経験が浅い人でも使えること
- PIIを収集しないプライバシーファーストな設計

## 4. 非目標（このキットに含まれないもの）

- メールマーケティング機能（Mailchimp等との連携）
- 購入者管理機能（CRM）
- コンテンツ保護DRM
- 決済の内製化（Stripeを直接実装する場合を除く）
- 多言語対応の自動化
- 動画コンテンツのホスティング
- 法務・税務アドバイス（テンプレートのみ提供）
- 収益・売上の保証

## 6. アーキテクチャ概要

```
購入者ブラウザ
    │
    ├─ GET / → Astro静的LP（CDN配信）
    ├─ GET /p/{slug} → 商品詳細ページ（CDN配信）
    ├─ GET /go?product=X&channel=Y&ref=Z
    │       └─ Cloudflare Pages Function
    │               ├─ KV書き込み（クリック計測）
    │               └─ 302 Redirect → 販路URL
    └─ GET /stats?days=14
            └─ Cloudflare Pages Function（ADMIN_TOKEN認証）
                    └─ KV読み込み → HTMLレポート

設定フロー（オーナー）:
config/products.yaml
    └─ pnpm gen → src/generated/products.json
            └─ pnpm build → dist/（Astro静的ファイル）
                    └─ GitHub Push → Cloudflare Pages自動デプロイ
```

---

## 7. 開発方針

### データフロー
1. `config/products.yaml` が唯一の真実（Single Source of Truth）
2. `pnpm gen` でJSONに変換・バリデーション
3. Astroビルド時にJSONを読み込みページ生成
4. `functions/go.ts` はKV/env経由でproduct情報を参照

### セキュリティ方針
- オープンリダイレクト防止: 遷移先URLはproducts.yamlに登録したもののみ
- PIIゼロ: IPアドレス・ユーザーエージェントは保存しない
- /stats はAdmin Token認証必須
- セキュリティヘッダーは `public/_headers` で一元管理

### 拡張性
- 商品追加: `products.yaml` に1エントリ追加するだけ
- 販路追加: `ChannelsSchema` と `getActiveChannels` を拡張
- 計測粒度向上: KVキー構造を変更して時間別集計も可能

## 10. リスクと対策

| リスク | 影響 | 対策 |
|--------|------|------|
| Cloudflare仕様変更 | KV API変更でデータ失損 | KVデータのバックアップスクリプト用意 |
| 販路手数料変更 | 収益計算がずれる | config/fees.yamlを定期更新 |
| 法的要件の変化 | 特商法等の改正 | 定期的に専門家確認 |
| KV無料枠超過 | クリック計測が止まる | Workers Paidへの移行を検討 |

---

### 作った背景

デジタルコンテンツを販売しようとすると、こんな問題にぶつかりませんか？

- どの販路がいいかわからない
- LP作るのが大変
- クリックを計測したいけどGA入れると個人情報が…
- 出品原稿を毎回ゼロから書くのが辛い

これらを全部解決するキットを作りました。

---

### できること

1. **SEO対応LP** — Astroで静的生成。JSON-LD・OGP・canonical 自動対応。
2. **5販路への購入導線** — BOOTH / note / Gumroad / BASE / Stripe を一元管理。
3. **クリック計測** — 個人情報なしでチャネル別クリックをKVに記録。
4. **出品原稿生成** — `pnpm gen:marketplace` で各販路向け原稿を一発生成。
5. **SNS投稿テンプレ** — X・note向けを30本分自動生成。

## 記事3: 各販路の使い分け

**タイトル**: 「BOOTH・note・Gumroad・BASE・Stripe どれを使うべきか」

（docs/CHANNEL_PLAYBOOK.md の内容を記事化する）

### BOOTH

- 同人・クリエイター向け
- pixiv連携でフォロワーへの告知が容易
- 手数料22%（変更の可能性あり）
- 日本語圏に強い

### note

- 記事と販売が一体化
- SEO流入を活かしやすい
- 既存フォロワーへの告知がしやすい
- 手数料10%程度（変更の可能性あり）

### Gumroad

- 英語圏向けに強い
- Discoverタブ経由は手数料30%に注意
- 直リンクは10%
- 海外クリエイターとのコラボに向く

### BASE

- 国内EC向け
- デジタルコンテンツ配布機能あり
- サービス利用料3%+決済手数料3.6%

### Stripe

- 手数料が最も低い（3.6%〜）
- 自前LPと組み合わせると高粗利
- 設定がやや複雑
